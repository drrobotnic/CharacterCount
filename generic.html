More on Report an Issue

If there is an issue, report to my repo

Method 1

The best - and probably the most helpful - option there is is to report to my GitHub repo and then we can work out the issue. Remember, this is an open-source project, so if you are a code or programmer, you can make your own changes to Skele-ton. If you are not a coder, hang tight. We will get to the issue.

There is also just sending me a message on my social media accounts, but this should only be neccesary when there is a bug that is user-ciritical. If this is not the case, it is best to just go to the GitHub repo and post your issue there. It would help, and would also gain more traction there

Method 2

If Method 1 does not work, or you would like to try this method, you can also just sit back and be patient. There is a chance that we will prbably catch the issue. If it is a user-reproducable bug, it is just best to post on the issues page on the GitHub repo.
